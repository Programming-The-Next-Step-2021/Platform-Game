Introduction¶

Platform Game is a package that uses mostly pygame to create a game, which can then be played. For creating a game, various things need to be programmed to make it work.
For example, a screen must be displayed, in here the game and players must be seen, key presses must be processed which allows for player movement, etc.

The most common imported functions & classes used are:

  • pygame.sprite.Sprite: A simple base class for visible game objects, used to create Classes (e.g., enemy class)
  • pygame.Rect: A pygame object for storing rectangular coordinates, which was used to get location of players and enemies.
  • pygame.Rect.colliderect: Tests if two rectangles overlap, used to program player object interactions (e.g., walking into a wall).

Most important created classes are:

  • Fighter(): See fighter.py section for more info.
  • HealthBar(): See player_attributes.py section for more info.
  • Item(): See terrain_objects.py section for more info.
  • TerrainObject(): See terrain_objects.py section for more info

How to start the game¶

To start the game, one only has to run the main.py file, see image underneath:

Once the game starts, press the start button to start the game.
    

To move use the arrow keys: the up arrow to climb up ropes (unfortunately I was not able to create this functionality yet, but will do so in the future),
down arrow to climb down ropes (idem ditto), left arrow to move left, right arrow to move right and spacebar to jump.
The goal is to reach the exit of the level. Whilst moving there, enemies will be on the road. Use a to slice
enemies with your sword. If you have taken too much damage, look for healthboxes to regenerate! Just walking over them
will suffice.

If you want to know more besides how to start the game, take a look at the following section in which the structure
of the code is explained.

Files¶

  Code Folder

  This folder contains all the code and files necessary to run the game

$\qquad$Audio folder¶

$\qquad$This folder contains audio for the game. These include whole songs and soundtracks (e.g., jump sound).
$\qquad$All songs are maplestory songs. The intro song is the maplestory theme song, the lvl2 song is the Kerning City: bad guys soundtrack,
$\qquad$and the finish song is the Sleepywood soundtrack.

$\qquad$Img folder¶

$\qquad$This folder contains seperate folders of images that need to be shown on the screen.
$\qquad$For example, the folder player contains folders with images of the player moving left, moving right, or jumping, etc.
$\qquad$There is also a background folder, with the image of the background.There are also folders with images for platforms,
$\qquad$water, item boxes, etc. Images were taken from: https://www.spriters-resource.com/pc_computer/maplestory/.

$\qquad$Level_data folder¶

$\qquad$The level_data folder contains 1 or more csv files, which provide information about the level setup (1 csv file per lvl).
$\qquad$In the csv file, in each cell, there are certain numbers that each correspond to a tile of a certain type.
$\qquad$For example, 0 = nothing, 1 = 'ground tile', 2 = 'water tile', etc.
$\qquad$Each number (tile), has an image (in the tile folder) that goes along with it.\ $\qquad$By looping over the numbers in the csv file, images will be loaded for certain numbers.\ $\qquad$This way, the position of the number (tile) in the csv file decides what image is going to be drawn where on the screen.

$\qquad$Loading_spritesheets.py¶

$\qquad$In this script, a spritesheet is given as input:

$\qquad$

$\qquad$A spritesheets contains different images for a game character, when doing different actions.
$\qquad$For example, the first row is for standing still, the second row for moving, the third for being hit and the fourth for dying.
$\qquad$This scripts loads the spritesheet, and creates and saves new image files for each seperate image.These will then be used in the game,

$\qquad$In this file, the class: HealthBar(), is introduced. This class basically creates
$\qquad$the healthbar for the player and updates it dependend on the health of the player.
$\qquad$For example, when a player is damaged, a part of the normally green healthbar will
$\qquad$turn red, indicating that you have taken damage.

$\qquad$Terrain_objects.py¶

$\qquad$In this file, various classes are introduced that creates certain level decorations or items.
$\qquad$- Item(): creates items to be picked up. For example, health. Once they are picked, the item is deleted.
$\qquad$- TerrainObject(): A base class that contains methods to ensure that certain objects move relatively with the player
$\qquad$ (e.g., when player moves left, the Item moves relatively with the player). This base class can create multiple objects:
$\qquad$$\qquad$- Decorations, like grass or Rocks.
$\qquad$$\qquad$- Water
$\qquad$$\qquad$- An Exit, to proceed to the next level. This happens once contact with the exit is made.

$\qquad$World.py¶

$\qquad$In this file the class World() is shown. This class basically creates the whole world with all the objects
$\qquad$(players, enemies, decoration, healthbar, items, etc) in it, which will be used in the main.py.
$\qquad$The most important function in this class is def process_data() which loops through all the numbers (tiles)
$\qquad$in the csv file. For a certain number, a certain thing is created (e.g., 9 or 10 = water). See example underneath:

def process_data(self, data: list[list[int]], img_list: list[pygame.Surface], item_boxes: dict[str, pygame.Surface])\
        -> (Fighter, Group, Group, Group, Group, Group):
    """ Processes the data of the world into usable images to draw on the screen.

    :param data: A csv file with integers that correspond to images to create the world with
    :param img_list: The images that correspond to the number in the data file
    :param item_boxes: A dictionary with the type of box and image of the box in it
    :return: A tuple containing: player, enemy_group, decoration_group, water_group, item_box_group, exit_group
    """

    # Create groups of sprite
    enemy_group = pygame.sprite.Group()
    decoration_group = pygame.sprite.Group()
    water_group = pygame.sprite.Group()
    item_box_group = pygame.sprite.Group()
    exit_group = pygame.sprite.Group()

# iterate through every value in the level data file which corresponds to images
        for y, row in enumerate(data):  # go through rows of the file
            for x, tile in enumerate(row):  # go through tiles of the row
                if tile >= 0:  # ignore -1s
                    img = img_list[tile]  # tile at position x will become the tile from the image list made earlier
                    img_rect = img.get_rect()
                    img_rect.x = x * TILE_SIZE  # x coordinate of where you put the rectangle of te the image
                    img_rect.y = y * TILE_SIZE  # same but for y coordinate
                    tile_data = (img, img_rect)  # tuple with info about tile: the image and the position
                    if 0 <= tile <= 8:  # 0 to 8 are the tiles that are dirt blocks thus obstacles
                        self.obstacle_list.append(tile_data)
                    elif 9 <= tile <= 10:  # this is water
                        water = TerrainObject(img, x * TILE_SIZE, y * TILE_SIZE)
                        water_group.add(water)
                    elif 11 <= tile <= 14:  # decorations, pile of rocks, wooden box, grass, etc
                        decoration = TerrainObject(img, x * TILE_SIZE, y * TILE_SIZE)
                        decoration_group.add(decoration)
                    elif tile == 15:  # create player
                        # select 'player' image, positioned at tile position * size, thus dependend on the lvl size *
                        # 0.2 and speed of 5
                        player = Fighter('player', x * TILE_SIZE, y * TILE_SIZE, PLAYER_SCALE, PLAYER_SPEED,
                                         facing_left=True)
                    elif tile == 16:  # create enemies
                        # select 'enemy' image, at tile position * size, image size 1 and speed of 5
                        enemy = Fighter('enemy', x * TILE_SIZE, y * TILE_SIZE, ENEMY_SCALE, ENEMY_SPEED,
                                        facing_left=True)
                        enemy_group.add(enemy)
                    elif tile == 19:
                        item_type = 'Health'
                        image = item_boxes[item_type]
                        item_box = Item(image, item_type, x * TILE_SIZE, y * TILE_SIZE)
                        item_box_group.add(item_box)
                    elif tile == 20:  # create exit
                        exit = TerrainObject(img, x * TILE_SIZE, y * TILE_SIZE)
                        exit_group.add(exit)
                    elif tile == 35:  # End bosss
                        # select 'enemy' image, at tile position * size, image size 1 and speed of 5
                        enemy = Fighter('boss_enemy', x * TILE_SIZE, y * TILE_SIZE, BOSS_ENEMY_SCALE, ENEMY_SPEED,
                                        facing_left=True)
                        enemy_group.add(enemy)
                    # all other tiles are all obstacles:
                    else:
                        self.obstacle_list.append(tile_data)

 LevelEditor folder

 This folder contains a program to create maps for the game and was created by Russ123.
 In the level editor, basically a grid of lines is created over the defined screen size,
 thus creating small squares over the whole screen. Per square, an image will be placed.
 This way, a map can be created.

 The level_editor_tut.py contains the main file, which when run simulates the level editor.
 The folder img contains the background folder, containing images for the background, the tile folder, which contains
 images of tiles and images for the load and save button. The first 20 images of the tile folder, were originally there.
 The other 20+ images were added by me, so that more different kinds of maps could be created.
 The original code was adapted slightly to make the functionality for more images possible.
 The images were all taken from the maplestory_tile_spreadsheat, which were created by Mageker

 How to use  If you run the main file, the program will start. Click the tile of preference on the right panel,
 and click on the map to apply it. Once finished, click save. The map will be saved as a csv file,
 for which certain numbers correspond to the images used per square (tile). This csv file can then
 be used in the platform game itself. Just copy paste it to the level_data folder in the Platform_Game folder.
  A csv file can also be loaded if it exists in the LevelEditor folder, and it is named: level(number)_data (e.g., level0_data).
